Technical architecture and design decisions for the Stencil Design System monorepo.

## StencilJS Architecture

### Component Compilation

Stencil compiles components to:
1. **Web Components** - Standard custom elements
2. **Framework Wrappers** - React, Vue, Angular components
3. **Type Definitions** - TypeScript `.d.ts` files

### Output Targets

Configured in `stencil.config.ts`:

```typescript
export const config: Config = {
  outputTargets: [
    // Standard web components
    {
      type: 'dist',
      esmLoaderPath: '../loader',
    },
    
    // React wrappers
    reactOutputTarget({
      componentCorePackage: '@stencil-kit/core',
      proxiesFile: '../@stencil-kit/react/lib/components/stencil-generated/index.ts',
    }),
    
    // Vue wrappers
    vueOutputTarget({
      componentCorePackage: '@stencil-kit/core',
      proxiesFile: '../@stencil-kit/vue/lib/components.ts',
    }),
    
    // Angular wrappers
    angularOutputTarget({
      componentCorePackage: '@stencil-kit/core',
      outputType: 'standalone',
      directivesProxyFile: '../angular-workspace/projects/component-library/src/lib/stencil-generated/components.ts',
    }),
  ],
};
```

### Shadow DOM

All components use Shadow DOM for:
- **Style encapsulation** - Styles don't leak
- **DOM encapsulation** - Internal structure hidden
- **Consistent rendering** - Same across frameworks

## Monorepo Management

### Lerna + Nx

**Lerna** handles:
- Package versioning
- Publishing to npm
- Running commands across packages

**Nx** provides:
- Task orchestration
- Build caching
- Dependency graph analysis
- Parallel execution

### Task Pipeline

Defined in `nx.json`:

```json
{
  "targetDefaults": {
    "build": {
      "dependsOn": ["^build"],
      "cache": true,
      "outputs": ["{projectRoot}/dist"]
    }
  }
}
```

**How it works:**
1. `^build` means "build dependencies first"
2. `cache: true` enables Nx caching
3. `outputs` defines what to cache

### Build Flow

```
User runs: npm run build
    ↓
Lerna identifies packages to build
    ↓
Nx creates task graph
    ↓
Nx checks cache
    ↓
If cached: Skip build
If not cached:
    ↓
    Build @stencil-kit/core
        ↓
        Generate wrappers
            ↓
            Build @stencil-kit/react, @stencil-kit/vue, angular-workspace (parallel)
                ↓
                Cache outputs
```

## Build System Details

### TypeScript Configuration

**Root `tsconfig.json`:**
```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true
  }
}
```

**Package `tsconfig.json`** (extends root):
```json
{
  "extends": "../../tsconfig.json",
  "compilerOptions": {
    "rootDir": "./lib",
    "outDir": "./dist",
    "paths": {
      "@stencil-kit/core": ["../stencil-library/dist/types"]
    }
  }
}
```

### Package.json Configuration

**Stencil Library:**
```json
{
  "main": "./dist/index.cjs.js",
  "module": "./dist/index.js",
  "types": "./dist/types/index.d.ts",
  "exports": {
    ".": {
      "import": "./dist/index.js",
      "require": "./dist/index.cjs.js",
      "types": "./dist/types/index.d.ts"
    },
    "./dist/*": "./dist/*"
  }
}
```

**Wrapper Libraries:**
```json
{
  "main": "./dist/index.js",
  "types": "./dist/index.d.ts",
  "files": ["dist/"],
  "dependencies": {
    "@stencil-kit/core": "*"
  }
}
```

## Caching Strategy

### Nx Cache

Nx caches:
- Build outputs (`dist/` folders)
- Task results
- Terminal output

**Cache Key** based on:
- Source file changes
- Dependency changes
- Configuration changes

**Benefits:**
- Faster rebuilds (skip unchanged packages)
- Consistent builds
- CI/CD optimization

### Cache Location

```
node_modules/.cache/nx/
```

### Clear Cache

```bash
npx nx reset
```

## Performance Considerations

### Build Performance

- **Nx caching** - Skip unchanged packages
- **Parallel execution** - Build independent packages simultaneously
- **Incremental builds** - Only rebuild changed files

### Runtime Performance

- **Lazy loading** - Components load on demand
- **Tree shaking** - Remove unused code
- **Shadow DOM** - Efficient style scoping
- **Small bundles** - Minimal overhead

### Bundle Sizes

Approximate sizes (gzipped):
- `my-button`: ~2KB
- `my-card`: ~3KB
- `my-badge`: ~1.5KB
- Total library: ~15KB

## Design Decisions

### Why StencilJS?

- **Framework agnostic** - Works everywhere
- **Web standards** - Built on web components
- **TypeScript** - Type safety
- **Small runtime** - Minimal overhead
- **Great DX** - Excellent developer experience

### Why Monorepo?

- **Code sharing** - Shared components and utilities
- **Atomic changes** - Change multiple packages together
- **Consistent tooling** - Same build system everywhere
- **Easier testing** - Test integrations locally

### Why Lerna + Nx?

- **Lerna** - Mature monorepo tool, great for publishing
- **Nx** - Modern task runner with caching
- **Best of both** - Lerna for versioning, Nx for speed

### Why Shadow DOM?

- **Style encapsulation** - No CSS conflicts
- **Framework compatibility** - Works in all frameworks
- **Maintainability** - Easier to reason about styles
